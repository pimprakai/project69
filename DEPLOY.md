# คู่มือการนำแดชบอร์ดขึ้นระบบเผยแพร่บน GitHub Pages

ไฟล์ชุดนี้ถูกออกแบบมาเพื่อเป็นหน้าเว็บแบบ Static (HTML/CSS/JS) ซึ่งมีข้อดีคือสามารถเผยแพร่บน **GitHub Pages** ได้ฟรี สะดวก และปลอดภัย โดยระบบจะดึงข้อมูลแบบ Real-time จาก Google Sheet โดยตรงทุกครั้งที่มีผู้เข้าใช้งานหน้าเว็บ

ทำตามขั้นตอนด้านล่างนี้เพื่อเปิดใช้งานเว็บไซต์ของท่าน:

---

## ขั้นตอนที่ 1: ติดตั้ง Git และเตรียมไฟล์บนคอมพิวเตอร์ของคุณ

หากคอมพิวเตอร์ยังไม่มี Git ให้ดาวน์โหลดและติดตั้งจาก [git-scm.com](https://git-scm.com/)

1. เปิดเครื่องมือ Terminal หรือ Command Prompt (cmd) ขึ้นมาแล้วเข้าไปยังโฟลเดอร์ของโปรเจกต์นี้:
   ```bash
   cd d:/20-6-69
   ```
2. เริ่มต้นระบบ Git ในโฟลเดอร์ด้วยคำสั่ง:
   ```bash
   git init
   ```
3. สร้างไฟล์ `.gitignore` (ไม่บังคับ แต่แนะนำ) เพื่อข้ามไฟล์ที่ไม่เกี่ยวข้อง:
   ```bash
   echo "node_modules/" > .gitignore
   ```
4. ทำการบันทึกไฟล์ทั้งหมดเข้าสู่ประวัติ Git (Commit):
   ```bash
   git add .
   git commit -m "Initial commit: Baan Pa Thua School Dashboard"
   ```

---

## ขั้นตอนที่ 2: สร้าง Repository บน GitHub

1. เข้าไปที่เว็บไซต์ [github.com](https://github.com/) และเข้าสู่ระบบบัญชีของท่าน
2. คลิกปุ่ม **"New"** (หรือไอคอนเครื่องหมายบวก `+` ที่มุมขวาบน เลือก **New repository**)
3. ตั้งชื่อ Repository เช่น `baanpathua-dashboard`
4. ตั้งค่าระดับการเข้าถึงเป็น **Public** (จำเป็นสำหรับบริการ GitHub Pages เวอร์ชันฟรี)
5. ปล่อยการตั้งค่าอื่นๆ (Add README, .gitignore, license) ให้เป็นค่าว่างทั้งหมด
6. คลิกปุ่ม **"Create repository"**

---

## ขั้นตอนที่ 3: อัปโหลดโค้ดขึ้น GitHub

ในหน้าจอหลักของ Repository บน GitHub ที่สร้างเสร็จใหม่ๆ จะแสดงชุดคำสั่งสำหรับดันโค้ด ให้คัดลอกมาวางใน Terminal ของคอมพิวเตอร์ท่าน หรือใช้คำสั่งด้านล่างนี้ (แทนที่ `YOUR-USERNAME` ด้วยชื่อผู้ใช้ GitHub ของท่าน):

```bash
# 1. เปลี่ยนชื่อสาขาหลักเป็น main
git branch -M main

# 2. เชื่อมโยงโฟลเดอร์กับ Repository บน GitHub
git remote add origin https://github.com/YOUR-USERNAME/baanpathua-dashboard.git

# 3. ดันไฟล์ทั้งหมดขึ้นไปยัง GitHub
git push -u origin main
```
*หมายเหตุ: ในการรันคำสั่ง `git push` ครั้งแรก ระบบอาจจะเปิดเบราว์เซอร์ให้ท่านกดยืนยันตัวตนกับทาง GitHub*

---

## ขั้นตอนที่ 4: เปิดใช้งาน GitHub Pages

เมื่อดันโค้ดขึ้นไปแล้ว ให้เข้าไปที่หน้าเว็บ Repository ของท่านบน GitHub:

1. คลิกที่เมนู **"Settings"** (แท็บรูปฟันเฟืองด้านบน)
2. เลื่อนเมนูด้านซ้ายไปที่หัวข้อ **"Code and automation"** -> เลือก **"Pages"**
3. ภายใต้หัวข้อ **Build and deployment**:
   - ในช่อง **Source** ให้ตั้งค่าเป็น **"Deploy from a branch"**
   - ในช่อง **Branch** ให้เลือกเป็น **`main`** และโฟลเดอร์เป็น **`/ (root)`**
4. คลิกปุ่ม **"Save"**
5. รอระบบของ GitHub ทำการประมวลผลประมาณ 1-2 นาที ให้ลองรีเฟรชหน้าการตั้งค่า Pages นั้นอีกครั้ง 
6. ท่านจะพบแถบข้อความแจ้งเตือนสีเขียวที่ระบุลิงก์เว็บไซต์ของท่าน เช่น:
   **"Your site is live at `https://YOUR-USERNAME.github.io/baanpathua-dashboard/`"**

เมื่อได้ลิงก์แล้ว ผู้บริหารโรงเรียนบ้านป่าถั่วจะสามารถเข้าเยี่ยมชมและเปิดใช้งานแดชบอร์ดจากอุปกรณ์ใดก็ได้ และระบบจะอัปเดตข้อมูลสดโดยตรงจาก Google Sheet ทุกครั้งที่เปิดหน้าเว็บ!
